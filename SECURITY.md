# 安全策略

## 支持范围

安全修复优先应用到最新 `master` 和最新阶段标签。早期阶段标签用于审计开发历史，不保证单独维护。

## 报告漏洞

请通过仓库所有者 GitHub 主页提供的私密联系方式，或 GitHub Private Vulnerability Reporting（启用后）报告。不要在公开 Issue、讨论区或 Pull Request 中披露尚未修复的漏洞。

报告应包含：

- 受影响版本或提交；
- 复现前提和最小步骤；
- 实际影响；
- 建议修复（如有）。

请勿在报告中附带真实 Token、密码、Cookie、私钥、个人数据或未经脱敏的数据库。

## 安全边界

- 当前版本没有用户登录，默认只监听 `127.0.0.1`。
- 不应在未增加认证、TLS、CSRF 和访问控制前直接暴露到公网。
- 系统只采集公开且允许自动访问的内容，不绕过登录、验证码、付费墙或 robots.txt。
- 额外 CA 必须由部署者明确提供；系统不支持关闭证书校验。

一般使用问题请查看[FAQ](docs/09-faq/faq.md)。
