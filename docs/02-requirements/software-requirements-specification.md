# 软件需求规格说明书（SRS）

## 1. 范围与约束

本规格定义 Knowledge Collector 第一版本的功能、质量、安全、数据与接口需求。系统为 Java 17/Spring Boot 3 模块化单体，本地单用户部署，使用 H2 文件数据库、Flyway、Thymeleaf 和本地文件存储。

## 6. 接口要求

- 页面路由与 REST Controller 分离，Controller 不承载业务规则。
- REST API 统一前缀 `/api/v1`，JSON 响应包含成功标志、数据或错误码、消息和时间。
- 列表 API 支持页码、每页数量、排序与过滤参数，并限制最大页大小。
- 创建和修改请求使用 DTO 与 Bean Validation。
- 外部 HTTP、RSS、HTML、存储、分类和搜索通过接口解耦。
- 需要幂等的状态切换和重试操作应防止重复执行。

## 7. 异常流程

| 场景 | 预期处理 |
| --- | --- |
| URL 非法 | 拒绝保存或任务项失败，返回明确校验错误 |
| robots.txt 禁止 | 跳过采集，记录合规原因 |
| 401/403 | 不尝试绕过，记录并停止 |
| 429/5xx | 按配置有限退避，耗尽后失败 |
| 响应超限或类型不符 | 中止读取，记录错误码 |
| 页面依赖复杂 JavaScript | 标记 `UNSUPPORTED_DYNAMIC_PAGE` |
| 正文为空/过短 | 跳过或进入待审核 |
| 重复 URL/正文 | 不创建新文章，更新采集统计 |
| 数据库/文件写入失败 | 事务回滚或补偿，任务记录失败 |

## 8. 边界条件

- 单个 HTML 响应建议最大 5 MB；单张图片建议最大 10 MB；最大重定向 5 次。
- 请求间隔默认不少于 2000 ms，最大重试次数必须为有限非负整数。
- 质量分限定 0—100；分页大小设置上限。
- 所有可输入文本设置合理长度；笔记与正文大字段单独限制。
- 不应删除所有查询参数，只移除明确的跟踪参数。
