# 产品需求文档（PRD）

## 1. 产品定位

Knowledge Collector 是面向个人与小团队的本地资料采集、清洗、分类和阅读管理工具。产品强调配置化、可观察、可控和合规，不定位为通用高并发爬虫平台。

## 2. 产品目标

- 降低定期获取多来源内容的人工成本。
- 降低重复、广告和危险 HTML 对阅读的干扰。
- 建立从采集到阅读行动的完整闭环。
- 让失败、限速和来源健康状况可见、可诊断。
- 保证本地部署、数据可控和无付费服务强依赖。

## 3. 核心指标

| 指标 | 第一版目标 |
| --- | --- |
| 核心流程完成率 | 用户可完成端到端闭环 |
| 固定测试样例采集成功率 | 100% |
| 自动化测试网络依赖 | 0 |
| 重复文章新增率 | 对相同 URL/正文样例为 0 |
| 危险脚本保留率 | 对安全测试样例为 0 |
| 普通列表查询响应 | 目标 2 秒内（本机、1 万篇规模基准） |
| 启动外部服务依赖 | 0 |

指标是设计和验收目标，Stage 1 尚未执行性能或功能验证。

## 5. 核心业务流程

```mermaid
flowchart LR
    A[创建主题] --> B[配置采集源]
    B --> C[配置或测试规则]
    C --> D[手动或定时采集]
    D --> E[发现与标准化 URL]
    E --> F[提取并清洗正文]
    F --> G[去重、分类、评分]
    G --> H[资料库]
    H --> I[搜索与阅读]
    I --> J[收藏、已读、归档、笔记]
```

## 6. 页面结构

- 仪表盘：首页指标、最近任务、最近文章、主题分布和来源成功率。
- 主题：列表、新建、编辑、启停。
- 采集源：列表、新建、编辑、测试、执行采集。
- 采集规则：版本列表、编辑、复制、测试预览。
- 任务：任务列表、详情、取消、失败重试。
- 资料库：文章列表、组合筛选、搜索和批量状态操作。
- 阅读详情：正文、元数据、上一篇/下一篇、收藏和笔记。
- 设置与运维：调度、请求限制、数据目录、备份与日志。

## 7. 功能优先级

### P0

主题/来源管理、RSS/Atom、任务日志、URL 与内容去重、文章入库、资料库、阅读详情、本地持久化。

### P1

HTML 规则、正文安全清洗、主题匹配、质量评分、组合搜索筛选、收藏/已读/归档/笔记、仪表盘、定时任务和备份。

### P2

增强 JSON 映射、深色模式、文章目录、本地 Lucene、机器学习或大模型分类 Provider。

## 8. MVP 范围

MVP 对应 Stage 3—8 的可运行业务闭环；Stage 9 补齐定时与运维，Stage 10 达到发布验收。P2 能力不是第一版发布阻塞项。

## 9. 异常与空状态

- 无主题/来源：引导创建，禁止无意义采集。
- 来源测试失败：显示状态码、错误码和可操作建议。
- 动态页面：显示不支持，不尝试绕过。
- robots.txt 禁止：跳过并记录原因。
- 429/临时错误：有限退避；超过次数后失败。
- 正文为空或质量过低：进入待审核或记录跳过原因。
- 资料库为空：提供“创建来源/执行采集”入口。

## 10. 后续规划

- Spring Security 与多用户权限。
- 本地 Lucene 全文搜索。
- 可插拔浏览器 Provider（必须继续遵守合规边界）。
- 更丰富的导出、订阅和阅读统计。
- 可选的机器学习或大模型分类器，不作为系统启动依赖。

## 11. 需求验收条件

产品验收以 SRS 编号、测试证据和最终验收清单为准。核心路径必须在无外部付费服务下运行；安全清洗、有限重试、来源合规和数据持久化为发布阻断项。
