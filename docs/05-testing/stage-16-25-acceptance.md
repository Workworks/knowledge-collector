# Stage 16–25 验收说明

## 自动化覆盖

`Stage16To25KnowledgeWorkflowIntegrationTest` 从空 H2 数据库执行全部 12 个 Flyway 迁移，并通过真实 HTTP 请求验证：

- AI 推荐卡片创建、人工确认和卡片关系。
- 观点创建与带卡片来源的支持证据。
- 实体创建、别名实体合并。
- 事件、专题页、研究项目和研究资料。
- 带来源的综合归纳和写作草稿。
- 知识缺口、到期复习和下一次复习安排。
- 统计 API 与知识工作台页面渲染。

## 手工验收

1. 启动应用并访问 `/knowledge`。
2. 依次切换十个页签，创建对应知识资产。
3. 打开一篇已有正文的文章，选中片段并保存知识卡片。
4. 在文章页执行 AI 分析，检查完整结构化分析可展开且页面不溢出。
5. 使用 `http/stage-16-25-knowledge-workspace.http` 验证主要 API。
6. 完成一次到期复习，确认该记录从到期列表消失。

## 通过标准

- `mvn clean verify` 通过。
- 文档链接检查通过。
- 迁移可在空库和已有 V11 数据库上执行。
- 综合结论缺少来源时 API 返回业务校验错误。
