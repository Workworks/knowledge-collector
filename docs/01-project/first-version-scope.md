# 第一版本范围基线

## 1. 范围目标

第一版交付一个可在本地单机运行、面向单个管理员的完整资料采集与阅读应用，而不是单一爬虫示例。

## 2. 范围内

### 2.1 管理与配置

- 主题的创建、修改、启停、排序、关键词和排除关键词。
- 采集源的创建、修改、启停、测试和健康状态。
- RSS、Atom、HTML_LIST、JSON_API、MANUAL_URL 五类来源模型。
- HTML CSS Selector 规则配置、测试、复制和版本保留。

### 2.2 采集处理

- 手动采集、定时采集、单来源采集、按主题批量采集和有限失败重试。
- URL 标准化、URL 哈希、正文 SHA-256 去重。
- 正文提取、危险 HTML 清理、安全展示、摘要与阅读时长生成。
- 来源默认主题、关键词匹配、排除关键词和人工调整。
- 0—100 的可解释质量评分与低质量待审核。

### 2.3 资料库与运维

- 文章分页、排序、组合筛选与数据库模糊搜索。
- 阅读详情、收藏、已读/未读、归档、忽略、标签和个人笔记。
- 仪表盘、任务日志、失败原因、来源成功率。
- H2 文件持久化、本地内容目录、日志、启动脚本、数据备份和恢复说明。
- 自动化测试、Markdown 文档、Git 阶段提交和标签。

## 3. 范围外

- 多用户、注册登录、细粒度权限和多租户。
- 微服务、容器编排、分布式存储、消息队列和分布式任务调度。
- Elasticsearch、外部强依赖搜索服务或付费 API。
- 复杂 JavaScript 动态页面、登录态采集、验证码处理、代理池和反限制绕过。
- 自动下载第三方图片、视频或大规模二进制附件。
- AI/机器学习强制分类；只预留可替换接口。

## 4. MVP 优先级

| 优先级 | 能力 |
| --- | --- |
| P0 | 主题/来源配置、RSS/Atom、任务记录、文章入库、去重、资料库和本地持久化 |
| P1 | HTML 规则、正文清洗、主题匹配、搜索筛选、阅读状态、仪表盘 |
| P2 | JSON API 完整适配、深色阅读、长文目录、增强质量评分和本地 Lucene |

P2 不作为第一版发布阻塞项，除非后续阶段明确变更范围。

## 5. 发布边界

应用必须能够离线启动；联网只在执行外部采集时需要。若来源禁止自动访问、要求登录或依赖复杂脚本，系统应停止或标记不支持，而不是尝试绕过。
