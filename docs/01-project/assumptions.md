# 默认假设与待确认事项

## 1. 已采用的默认假设

1. 第一版本为本地单用户管理系统，只设置 `ADMIN`。
2. 第一版本不实现登录，但分层设计为后续 Spring Security 留出空间。
3. 默认界面语言为中文，默认时区为操作系统本地时区。
4. 只采集公开网页、RSS、Atom 和公开 API，不采集需要登录的内容。
5. 不绕过付费墙，不处理验证码，不使用代理池。
6. 不处理必须执行复杂 JavaScript 的页面，此类页面标记为 `UNSUPPORTED_DYNAMIC_PAGE`。
7. 默认检查并遵守 robots.txt。
8. 同一来源请求间隔默认不少于两秒，并限制同域并发。
9. 默认保存清洗后的正文和原文链接，不默认保存原始 HTML。
10. 默认不下载第三方图片到本地，只保留经过安全处理的图片链接。
11. 第一版本使用 H2 文件数据库和 Flyway。
12. 第一版本前端使用 Thymeleaf、Bootstrap 和原生 JavaScript。
13. 定时采集使用 Spring `@Scheduled`，不引入 Quartz。
14. 搜索使用数据库查询，不引入 Elasticsearch。
15. 文章默认按发布时间倒序；无法识别发布时间时使用采集时间并明确标记。
16. 同一文章可以属于多个主题。
17. 所有采集规则通过管理后台配置，不在代码中写死站点。
18. 初始演示数据只使用公开、稳定、适合测试的示例源。
19. 自动化测试不依赖实时外部网站。
20. 本地运行数据统一放在项目可配置的数据目录，代码不写死绝对路径。
21. 正文第一版使用混合存储：清洗 HTML、纯文本和摘要存入 H2 CLOB；可选原始 HTML 与页面快照存入本地文件，数据库只保存相对路径。
22. 无登录版本默认监听 `127.0.0.1:8080`；如管理员主动开放局域网访问，部署文档必须明确安全风险。
23. Maven 采用五模块模块化单体，依赖方向为 `domain → application → infrastructure/web → boot`。
24. JSON API 第一版只支持配置化的基础字段路径映射，不提供任意脚本转换能力。
