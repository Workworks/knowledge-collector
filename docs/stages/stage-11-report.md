# Stage 11 阶段报告

## 1. 阶段目标

进入第一版本后的维护迭代，修复 Windows 测试报告中文乱码和 RSS/Atom 文章缺少正文问题，提高来源兼容性与内容质量。

## 2. 本阶段范围

测试日志编码、RSS/Atom 内嵌全文处理、文章详情页通用正文抽取、安全清洗、空正文回填、阅读页空状态及回归测试。

## 3. 已完成事项

- 控制台日志编码跟随 JVM `file.encoding`，文件日志继续固定 UTF-8。
- Feed 配置把“允许采集正文”和“只保存摘要”传递到 Provider。
- RSS/Atom 优先使用 Feed 内嵌全文；内容不足时请求文章详情页。
- 通用正文抽取支持常见语义选择器及文本密度评分。
- 正文统一移除脚本、表单、导航、广告、评论、推荐等危险或噪声内容。
- 单篇详情页失败时回退到 Feed 内容或摘要，不拖垮整个 Feed 任务。
- 已有空正文文章再次采集时补写正文并重新执行质量评估。
- 阅读页为空时显示可操作提示，不再让来源说明看起来像正文。

## 4. 未完成事项

JavaScript 动态渲染、登录后内容、付费墙和站点访问控制不在本阶段范围内，也不会通过规避限制获取。

## 5. 新增文件

- 通用 HTML 文章正文提取器。
- 正文抽取单元测试。
- Stage 11 IDEA HTTP Client 验收请求。
- Stage 11 阶段报告。

## 6. 修改文件

Feed Provider、采集请求契约、任务服务、文章写入适配器、阅读模板、日志配置、README、FAQ、设计和测试文档。

## 7. 核心设计说明

正文抽取采用“明确语义选择器优先、候选节点评分回退”的策略。抽取结果必须经过 Jsoup Safelist 清洗；失败时只保存摘要和原文链接。正文补写只发生在数据库原正文为空且新结果非空时，避免覆盖用户已经采集的有效正文。

## 8. 数据库变更

无 Flyway 变更。复用现有 `content_html`、`content_text`、字数、阅读时长和评估数据。

## 9. 接口变更

对外 REST 路径和响应结构不变。内部 `FetchRequest` 增加全文与摘要策略字段。

## 10. 页面变更

阅读详情页增加“暂未采集到正文”空状态和重新采集说明。

## 11. 测试情况

覆盖中文超时提示、Feed 摘要模式、文章详情页正文提取、XSS/广告清洗、重复文章正文回填及阅读页展示。13 个测试套件、25 个测试，失败 0、错误 0、跳过 0。

## 12. 构建与运行结果

`mvnw clean verify` 成功，可执行 JAR 为 64,941,318 字节。健康状态为 UP，首页、资料库和 OpenAPI 均返回 HTTP 200，Flyway 版本为 V8。

## 14. 已知问题

通用正文抽取无法保证兼容所有网站。结构特殊的网站应使用 HTML 规则源配置精确 CSS Selector；动态页面仍只保留摘要和原文链接。

## 15. 风险与技术债务

开启 RSS 全文会增加对详情页的网络请求。后续可增加按域限速、正文抽取质量阈值配置和来源级选择器覆盖。

## 16. 验收结果

Stage 11 验收通过：自动化测试、中文报告检查和 JAR 冒烟均成功，两项用户反馈已形成回归测试。
