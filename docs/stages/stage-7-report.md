# Stage 7 阶段报告

## 阶段目标

完成主题规则匹配、排除控制、多主题分类、文章质量评分、内容指纹和待审核流程，并先修复浏览器自动请求 `/favicon.ico` 导致的控制台静态资源异常。

## 已完成

- 新增 `/favicon.ico` 专用端点，返回可缓存的 SVG 图标，不再触发静态资源未找到异常。
- Flyway V5 为文章增加内容指纹、质量分、审核状态、来源等级、证据数和评估提示，并新增文章主题关系表。
- 主题匹配同时考虑来源默认主题和主题关键词，标题、摘要、正文使用不同权重，支持一篇文章匹配多个主题。
- 排除词优先于默认主题和普通关键词；命中后文章进入待审核队列。
- 质量分综合标题、作者、发布时间、摘要、正文、主题匹配、HTTPS 和 DOI 等可解释信号。
- 来源等级支持 `PRIMARY_SOURCE`、`PREPRINT`、`DIRECT_SOURCE` 和 `AGGREGATED_METADATA`。
- 采集文章首次入库后自动评估；提供查询与手动重新评估接口。
- 文章列表支持按审核状态、最低质量分和主题过滤；新增 `/articles/review` 待审核页面。
- 文章详情页展示质量分、审核状态、来源等级、证据数、主题匹配原因和评估提示。
- IDEA HTTP Client 增加待审核、质量筛选、主题筛选、评估查询和重新评估请求。

## 接口与页面

- `GET /api/v1/articles?reviewStatus=PENDING_REVIEW`
- `GET /api/v1/articles?minQuality=60&topicId={topicId}`
- `GET /api/v1/articles/{articleId}/assessment`
- `POST /api/v1/articles/{articleId}/assessment`
- `GET /articles/review`
- `GET /favicon.ico`

## 验证

- 领域规则测试覆盖多主题、DOI 来源等级和排除词优先级。
- HTML 采集集成测试覆盖采集后自动评估、质量阈值、自动接收、主题关系和 64 位 SHA-256 指纹。
- 基础集成测试验证 favicon 返回 200 和 SVG Content-Type。
- 全量测试共 18 个，失败 0、错误 0。

## 已知边界

- 当前质量分是确定性启发式规则，不替代人工事实核查或学术同行评审。
- 待审核流程当前提供队列和重评能力，人工“批准/拒绝”操作随 Stage 8 阅读状态闭环补充。
- 内容指纹用于稳定识别与后续去重增强；当前 URL 唯一约束仍是入库去重的主要门禁。
- 主题规则变化后不会批量自动重算历史文章，当前可按文章调用重新评估接口。

## 下一阶段

Stage 8 实现资料库与阅读闭环：全文搜索与组合筛选、收藏、已读、归档、标签和笔记，并补齐待审核文章的人工处理动作。
