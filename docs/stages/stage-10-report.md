# Stage 10 阶段报告

## 1. 阶段目标

完成第一版本测试、部署和项目验收，并修复任务长期停留 RUNNING 与部分 RSS 证书链 PKIX 错误。

## 2. 本阶段范围

任务超时与租约、TLS 信任、备份恢复验证、测试计划与报告、部署和安装文档、用户与管理手册、FAQ、发布 JAR 验收。

## 3. 已完成事项

- HTTP 正文改为总请求超时覆盖的限长 BodySubscriber。
- 任务增加心跳、自动回收和手动回收入口。
- 任务完成使用状态条件，过期线程不能反写最终状态。
- Windows 合并系统根证书，并支持管理员配置附加 PEM CA。
- PKIX 错误转换为可操作的 `TLS-CERTIFICATE-UNTRUSTED`。
- 完成第一版测试、部署、安装、用户、管理和 FAQ 文档。
- 完成真实 H2 备份恢复验证。

## 4. 未完成事项

动态 JavaScript 页面、登录页面、分布式调度、专业搜索引擎和在线恢复不属于第一版本。

## 5. 新增文件

- Flyway V8、任务恢复调度器、TLS 上下文工厂。
- Stage 10 测试和运维文档。
- TLS 测试证书夹具和验收 HTTP 请求集。

## 6. 修改文件

采集 HTTP 客户端、任务网关与服务、运维 API/页面、配置、README 和集成测试。

## 7. 核心设计说明

同一来源仍由数据库唯一约束保证并发安全；心跳租约解决进程中断后的永久锁。TLS 使用组合 TrustManager，不提供信任所有证书的危险开关。

## 8. 数据库变更

Flyway V8 新增 `heartbeat_at` 和租约索引；升级时回收历史遗留的活动任务。

## 9. 接口变更

新增 `POST /api/v1/operations/tasks/recover-stale`。任务错误新增 `HTTP-REQUEST-TIMEOUT`、`TASK-TIMEOUT` 和 `TLS-CERTIFICATE-UNTRUSTED`。

## 10. 页面变更

运维页新增“回收超时任务”按钮。

## 11. 测试情况

12 个测试套件、23 个测试，失败 0、错误 0、跳过 0。覆盖慢响应、锁释放、租约回收、TLS CA、备份恢复和原有业务回归。

## 12. 构建与运行结果

`mvn clean verify` 成功。64,933,076 字节可执行 JAR 启动成功；健康、首页、运维页、回收接口和 favicon 验证通过。

## 13. Git 提交与标签

提交信息：`stage-10: 完成项目验收并修复任务超时与TLS证书`；标签：`stage-10`。

## 14. 已知问题

严重缺失中间证书且系统/JDK/附加 CA 均不信任的站点仍会被安全拒绝。管理员需要更新 JDK 或配置可信 CA。

## 15. 风险与技术债务

任务执行仍为同步单机模型；租约超时过短可能误回收超长处理，管理员应按数据规模配置。外部源格式和访问策略可能变化。

## 16. 验收结果

Stage 10 及第一版本验收通过，两个用户报告的阻断问题均有自动化回归证据。

## 17. 下一阶段计划

进入维护迭代：按实际使用反馈优化来源兼容、重试退避、搜索性能和可观测性，不再按原第一版阶段继续扩张范围。
