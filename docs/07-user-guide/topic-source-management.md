# 主题与采集源管理指南

## 1. 入口

启动应用后访问：

- 主题管理：`http://127.0.0.1:8080/topics`
- 采集源管理：`http://127.0.0.1:8080/sources`

首页也提供两个管理入口。

## 2. 主题管理

主题用于组织后续采集内容。可按名称或编码搜索，并按启用状态筛选。

创建主题时：

- 编码只允许字母、数字、下划线和连字符，保存时自动转为大写。
- 名称和编码均不能重复。
- 关键词与排除关键词可使用逗号、中文逗号或换行分隔，系统会去除重复项。
- 颜色使用 `#RRGGBB`。
- 停用主题不会自动删除历史配置。

如果主题已关联采集源，系统会阻止直接删除。请先编辑或删除相关采集源。

## 3. 采集源管理

支持配置 `RSS`、`ATOM`、`HTML_LIST`、`JSON_API`、`MANUAL_URL` 五种来源类型。

创建采集源时可以设置：

- 首页、订阅或 API 地址。
- 一个或多个已启用主题。
- 语言、字符集和 User-Agent。
- 超时时间、最大重试次数和请求间隔。
- robots.txt、正文、摘要和快照策略。

地址只允许 `http://` 或 `https://`。当前阶段只保存和管理配置；点击“测试”会明确提示该能力将在 Stage 5 由采集 Provider 提供，不会伪造成功结果。

## 4. 删除与停用

- 暂时不用的主题或来源优先选择停用。
- 删除来源会解除其主题关系。
- 已关联来源的主题不能直接删除。
- 后续产生文章后，来源删除将按受控数据保留策略扩展。

## 5. 接口测试

使用 IDEA 打开：

```text
http/stage-4-management.http
```

先执行创建主题，记录返回 ID；再把来源请求和详情请求中的示例 ID 替换为真实 ID。重复创建前修改编码和名称。
